# CBL YouTube Review Tool Ver1

大会練習後の反省会で、最大15人分のYouTube限定公開アーカイブを同じ基準時間で確認するための静的Webアプリです。

Conqueror's Bladeなど、各メンバーが別々のタイミングで配信開始した試合アーカイブを並べて、Discordの画面共有やVCで指示しながら見返す用途を想定しています。

## 起動方法

1. `index.html` をブラウザで開きます。
2. 各枠にプレイヤー名、YouTube URL、個別開始位置を入力します。
3. 「動画読み込み」を押します。
4. 必要に応じて「設定保存」または「JSON出力」を使います。

通常はローカルサーバーなしで動きます。

ブラウザや環境によって直接ファイルを開いたときにYouTube埋め込みが不安定な場合は、作業フォルダで簡易サーバーを起動してから開いてください。

```powershell
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000/` を開きます。

## YouTube URLの貼り方

以下の形式に対応しています。

- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/embed/VIDEO_ID`
- `https://www.youtube.com/live/VIDEO_ID`
- 余計なパラメータ付きURL
- 動画ID単体

URLを貼ったあと「動画読み込み」を押してください。

## 個別開始位置

各動画ごとに、試合の開始位置を入力します。

対応形式:

- `HH:MM:SS`
- `MM:SS`
- 秒数

例:

- `01:23:45`
- `23:45`
- `120`

不正な形式は0秒扱いになり、入力欄と状態表示で分かるようにしています。

## 反省会タイムの考え方

反省会タイムは、全動画共通の基準時間です。

各動画の実際の再生位置は以下になります。

```text
実再生位置 = 個別開始位置 + 反省会タイム
```

例:

- COMAの開始位置: `00:01:20`
- Player2の開始位置: `00:00:55`
- 反省会タイム: `00:05:00`

この場合、COMAは `00:06:20`、Player2は `00:05:55` に移動します。

## 基本操作

- 「全再生」: 読み込み済み動画を反省会タイムに再同期してから再生します。
- 「全停止」: 読み込み済み動画を停止します。
- 「5秒戻す」「5秒進める」: 反省会タイムを5秒移動して全動画を再同期します。
- 「10秒戻す」「10秒進める」: 反省会タイムを10秒移動して全動画を再同期します。
- 「ジャンプ」: 入力した反省会タイムへ移動します。
- 「再同期」: 現在の反省会タイムで全動画を合わせ直します。
- 「倍速一括変更」: 読み込み済み動画の再生速度をまとめて変更します。

倍速は `0.5 / 0.75 / 1.0 / 1.25 / 1.5 / 2.0` に対応しています。

## 音声操作

- 各枠の「ミュート解除」「ミュート」で個別に切り替えできます。
- 「全ミュート」で全動画をミュートします。
- 「全ミュート解除」で全動画のミュートを解除します。
- 「この動画だけ音声ON」で選択した動画だけ音声ONにし、他の動画をミュートします。

15本同時再生では音声が混ざるため、基本は「この動画だけ音声ON」を使うのがおすすめです。

## 拡大表示

各枠の「拡大表示」を押すと、その動画を大きく表示します。

カード内の入力欄やボタン以外の部分をクリックしても拡大表示できます。YouTubeプレイヤー本体はブラウザ仕様上クリックをページ側で拾えないため、確実に拡大したい場合は「拡大表示」ボタンを使ってください。

拡大中も全再生、全停止、再同期などの全体操作は使えます。

## 設定保存

「設定保存」を押すと、ブラウザのlocalStorageに保存します。

保存対象:

- プレイヤー名
- YouTube URL
- 個別開始位置
- ミュート状態
- 倍速
- 現在の反省会タイム

次回同じブラウザで開いたときに自動復元されます。

## JSON共有

「JSON出力」を押すと、現在の設定がテキスト欄に出ます。

使い方:

1. ホストが設定を作る。
2. 「JSON出力」を押す。
3. 表示されたJSONをDiscordなどで共有する。
4. 参加者はJSONを `.json` ファイルとして保存する。
5. 参加者は「JSON読み込み」から読み込む。

試合ごとの設定保存にも使えます。

## 共有リンク

「共有リンク作成」を押すと、現在の設定をURLのhashに入れた共有リンクを作成します。

共有リンクには15枠分のプレイヤー名、YouTube URL、個別開始位置、倍速などが含まれます。リンクが長くなりすぎる場合は画面に注意を表示します。

リンクがDiscordなどで途中で切れる場合は、JSON共有を使ってください。

## 注意点

- YouTubeの仕様上、完全なフレーム単位同期は保証できません。
- ズレた場合は「再同期」ボタンを押してください。
- 15本同時再生はPC負荷が高くなります。
- 重い場合はYouTube側の画質を下げてください。
- 不要な動画は一時停止する、拡大表示中心で確認するなどで負荷を下げられます。
- 動画投稿者側で埋め込み再生が無効になっている場合、このツール上で再生できない可能性があります。
- 限定公開URLの取り扱いには注意してください。

## トラブルシューティング

### 動画が表示されない

- URLが対応形式か確認してください。
- 限定公開URLが正しいか確認してください。
- 投稿者が埋め込み再生を無効にしている場合、このツール上では再生できません。
- YouTube側で年齢制限、地域制限、ログイン必須などがある場合、埋め込み再生できないことがあります。

### タイミングがズレる

- 「再同期」を押してください。
- 配信側の遅延やYouTubeプレイヤーの読み込み状態により、完全一致しないことがあります。
- 反省会タイムを直接入力して「ジャンプ」することもできます。

### PCが重い

- YouTubeプレイヤー側で画質を下げてください。
- 全員分を常に再生せず、必要な動画だけ再生してください。
- 音声は「この動画だけ音声ON」を使ってください。
- ブラウザの他タブや重いアプリを閉じてください。

### 設定が復元されない

- 同じブラウザで開いているか確認してください。
- ブラウザのlocalStorageが削除されていると復元できません。
- 重要な設定はJSON出力で別途保存してください。

## Ver2以降の候補

Ver1では、ホストが操作したら全員のブラウザも自動同期する機能は対象外です。

将来的には以下を検討します。

- WebSocketやWebRTCを使ったリアルタイム同期
- 部屋URLやセッションIDによる参加
- タイムラインメモ
- 試合ごとのプリセット管理
- 低負荷表示モード
