# CBL YouTube Review Tool Ver1.1

最大15本のYouTube限定公開アーカイブを、同じ反省会タイム基準で同時確認する静的Webアプリです。

Ver1.1では、設定担当者が動画URLと開始位置を入力して共有URLを作成し、そのURLを開いた人は設定画面を経由せずレビュー画面へ直行します。

## 起動方法

YouTubeエラー153を避けるため、`index.html` をダブルクリックして `file://` で開かないでください。

```powershell
cd C:\codex\cbl-youtube-review-tool
python -m http.server 5173
```

ブラウザで以下を開きます。

```text
http://localhost:5173/index.html
```

または:

```text
http://127.0.0.1:5173/index.html
```

GitHub Pages / Cloudflare Pages / Vercel などで公開すると、メンバーは公開URLを開くだけで使えます。APIキー、npm install、外部DBは不要です。

## 通常の使い方

1. 設定担当者が設定画面を開く。
2. 動画1〜動画15にYouTube URLを入力する。
3. 必要なら開始位置を `HH:MM:SS` または `MM:SS` で入力する。
4. 「共有URL作成」を押す。
5. 共有URLをDiscordなどで共有する。
6. メンバーが共有URLを開くと、レビュー画面へ直行する。

## 設定画面

設定画面は共有URLを作るための画面です。

- 最初から15枠すべて表示します。
- プレイヤー名は使いません。
- 空欄の動画は共有URLに含めません。
- 不正なURLは共有URLに含めません。
- 開始位置が不正な場合は0秒扱いで共有URLに含めます。
- 入力途中の内容は同じタブ内だけ `sessionStorage` に一時保存します。
- タブを閉じると入力途中の内容は消えます。
- `localStorage` には保存しません。

開始位置:

- OK: `01:23:45`
- OK: `23:45`
- OK: `00:01:20`
- OK: `01:20`
- 空欄: 0秒
- 警告: `abc` は0秒扱い
- 警告: `120` は0秒扱い

秒数のみの入力はVer1.1では不正形式です。

## 共有URL形式

共有URLは以下の形式です。

```text
https://公開先URL/#r=<URLエンコード済み設定データ>
```

エンコード前のpayload:

```text
1~再生速度~動画ID.開始秒,動画ID.開始秒
```

例:

```text
1~10~ABCDEFGHIJK.28,LMNOPQRSTUV.0,ZYXWVUTSRQP.2s0
```

共有URLでは `encodeURIComponent()` した値を `#r=` の後ろへ入れます。読み込み時は `decodeURIComponent()` します。

再生速度コード:

- `05` = 0.5倍
- `075` = 0.75倍
- `10` = 1.0倍
- `125` = 1.25倍
- `15` = 1.5倍
- `20` = 2.0倍

開始秒数は36進数です。例: 80秒は `28`、120秒は `3c`、3600秒は `2s0`。

共有URLに含めないもの:

- プレイヤー名
- 元のYouTube URL全文
- メモ
- ミュート状態
- 拡大状態
- 現在の再生位置
- 一時停止状態
- 設定画面の入力途中状態
- 画質希望

## レビュー画面

`#r=` 付き共有URLを開くと、設定画面を表示せずレビュー画面へ直行します。

- 有効な動画だけ表示します。
- 最大15本です。
- 初期状態は全ミュートです。
- 全再生を押すまでは停止状態です。
- 音声ONにできる動画は原則1本のみです。
- 拡大した動画は音声ONになり、他の動画はミュートされます。
- 動画の左上に `#1` 〜 `#15` を表示します。
- プレイヤー名は表示しません。
- エラー動画があっても、他の動画は止めません。

## 操作バー

レビュー画面の下部にコンパクトな操作バーがあります。

- 全再生
- 全停止
- 5秒戻す
- 5秒進める
- 30秒戻す
- 30秒進める
- 再同期
- 再生速度
- 全ミュート
- 設定編集

操作バーはレビュー画面を開いた直後に表示され、約3秒操作がなければ自動で隠れます。マウス移動やタップで再表示します。

## reviewTime

内部的に `reviewTime` を持ちます。

```text
実再生位置 = 各動画の開始位置 + reviewTime
```

再生中の `reviewTime` はsetIntervalだけで進めず、マスタープレイヤーの現在再生位置から算出します。

マスター優先順位:

1. 音声ON中の動画
2. 拡大表示中の動画
3. 再生可能な動画のうち最も若い番号の動画

YouTubeの仕様上、完全なフレーム単位同期は保証できません。ズレた場合は「再同期」を押してください。

## 拡大表示と音声

各動画の「拡大」ボタン、または動画枠のダブルクリックで拡大表示します。

- 拡大対象を中央に大きく表示します。
- 背景を暗くします。
- 他動画は裏で再生継続します。
- Escまたは閉じるボタンで戻ります。
- 拡大した動画はユーザー操作に連動して音声ONになります。
- 他動画は自動ミュートします。

YouTube iframe内の歯車やシークバー操作を邪魔しないため、透明レイヤーでiframeを覆う実装はしていません。

## 非表示

正常動画にもエラー動画にも「非表示」ボタンがあります。

- 押した動画はレビュー画面から除外されます。
- 全再生、全停止、秒移動、再同期の対象外になります。
- 可能な限りYouTube Playerを停止・破棄します。
- レイアウトは残り動画数で再計算します。
- Ver1.1では戻すボタンはありません。戻したい場合はページを再読み込みしてください。

すべて非表示にした場合は「表示中の動画がありません。戻すにはページを再読み込みしてください。」と表示します。

## JSON詳細機能

JSONは通常利用者向けではありません。

設定画面下部の「詳細機能: JSONバックアップ」を開いた場合だけ、JSON出力/読み込みが使えます。

用途:

- 設定担当者のバックアップ
- 後日の再編集
- 技術的に理解がある人向けの補助

通常運用では共有URLを使ってください。

## 画質について

ツール側から画質を一括指定・強制する機能は実装しません。

理由: YouTube IFrame Player APIでは、現在 `setPlaybackQuality` などによる画質制御は実質的に機能しません。

- 小さな画面サイズではYouTube側が自動的に低画質を選ぶ場合があります。
- 拡大時にはYouTube側が自動的に画質を上げる場合があります。
- 重い場合はYouTubeプレイヤー内の歯車アイコンから手動で画質を下げてください。

## 15本同時再生の注意

15本同時再生はPCスペック、GPU、メモリ、回線、ブラウザの状態に強く依存します。

環境によっては、一部動画が黒画面になる、カクつく、読み込みが止まる可能性があります。

安定運用する場合は、回線とPCが強いリーダーがツールを開き、Discordで画面共有する運用を推奨します。

重い場合は不要な動画を非表示にするか、YouTubeプレイヤー内の歯車から手動で画質を下げてください。

## エラー153対策

エラー153は、YouTubeにHTTP Refererまたは同等のクライアント識別情報が渡っていない場合に出ることがあります。

このツールでは以下を実装しています。

- `<meta name="referrer" content="strict-origin-when-cross-origin">`
- YouTube Player作成時に可能な場合 `origin: window.location.origin` を指定
- `file://` 起動時の警告表示

それでも `file://` では参照元情報を渡せない場合があります。localhostまたは公開URLから開いてください。

## ショートカット

ショートカットは補助機能です。効かない場合は操作バーを使ってください。

- Space: 全再生 / 全停止
- ←: 5秒戻す
- →: 5秒進める
- R: 再同期
- Esc: 拡大解除

入力欄にフォーカスがある場合は無効です。YouTube iframeにフォーカスがある場合、親画面側のショートカットが効かないことがあります。
