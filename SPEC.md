# CBL YouTube Review Tool Ver1 仕様

## 目的

Conqueror's Blade などの大会練習後に、最大15人分のYouTube限定公開アーカイブを同じ反省会タイム基準で並べて確認する。

Ver1は静的Webアプリとして、Discordの画面共有やVCで指示しながら使える状態を優先する。

## 対象範囲

- 最大15本のYouTube動画枠を表示する。
- 各枠にプレイヤー名、YouTube URL、個別開始位置を設定できる。
- YouTube IFrame Player APIを使って動画を埋め込み再生する。
- 反省会タイムを基準に、各動画を「個別開始位置 + 反省会タイム」へ同期する。
- 全再生、全停止、5秒/10秒戻す、5秒/10秒進める、任意タイムジャンプ、再同期を提供する。
- 倍速を一括変更する。
- 個別ミュート、全ミュート、全ミュート解除、選択中の動画だけ音声ONを提供する。
- クリックした動画を拡大表示できる。
- プレイヤー名、開始位置、実再生位置、ミュート状態、読み込み状態、エラー状態を表示する。
- 現在の反省会タイム、再生状態、倍速、読み込み済み動画数を表示する。
- localStorageへ設定を保存し、次回起動時に復元する。
- JSONで設定をエクスポート/インポートできる。
- URL hashを使った共有リンクを生成し、共有リンクから設定を復元する。
- 日本語UIで提供する。

## 対応YouTube URL

以下の形式から可能な限り動画IDを抽出する。

- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/embed/VIDEO_ID`
- `https://www.youtube.com/live/VIDEO_ID`
- 余計なクエリパラメータ付きURL
- 動画ID単体

## 開始位置形式

以下の形式を秒数へ変換する。

- `HH:MM:SS`
- `MM:SS`
- 秒数

不正な形式は0秒として扱い、UI上で警告表示する。

## 反省会タイム

反省会タイムは全動画共通の基準時間である。

各動画の実再生位置は以下で決まる。

`実再生位置 = 個別開始位置 + 反省会タイム`

例:

- COMA開始位置 `00:01:20`
- Player2開始位置 `00:00:55`
- 反省会タイム `00:05:00`

この場合、COMAは `00:06:20`、Player2は `00:05:55` へ移動する。

## レイアウト

- PCでは5列×3行を基本とする。
- 画面幅に応じて3列、2列、1列へ変化する。
- 各動画枠は入力欄、状態表示、操作ボタン、YouTubeプレイヤーを持つ。

## 対象外

Ver1では以下を対象外とする。

- ホスト操作を参加者ブラウザへリアルタイム同期する機能
- ユーザー認証
- YouTube APIキーを使う機能
- 外部データベース保存
- 外部サービスへのデプロイ
- GitHubへのpush
- 課金が発生する処理
- フレーム単位の完全同期保証

## 将来拡張候補

- WebSocketやWebRTCを使ったホスト操作のリアルタイム同期
- 部屋URL/セッションIDによる参加者同期
- 試合ごとのプリセット管理
- タイムライン上のメモ、タグ、クリップポイント
- 役割別レイアウト保存
- YouTube以外の動画ソース対応
- パフォーマンス優先の低負荷表示モード
- キーボードショートカット
- 動画の一括画質調整。ただしYouTube IFrame APIで制御できる範囲に制限される。

## 注意事項

- YouTubeの仕様上、完全なフレーム単位同期は保証しない。
- ズレた場合は「再同期」ボタンで補正する。
- 15本同時再生はPC負荷が高い。
- 重い場合はYouTube側の画質を下げる。
- 投稿者側で埋め込み再生が無効な動画は、このツール上で再生できない可能性がある。
- 限定公開URLの取り扱いには注意する。
